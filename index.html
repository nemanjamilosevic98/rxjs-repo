  • Join-Creation Operators
    • combineLatest

      Description:
      - Combines multiple Observables to create an Observable whose values are calculated from the latest values of each of its input Observables

      Marble diagram:
      (an example is in the console)
    • concat

      Description:
      - Creates an output Observable which sequentially emits all values from the first given Observable and then moves on to the next

      Marble diagram:
      (an example is in the console)
    • forkJoin

      Description:
      - Wait for Observables to complete and then combine last values they emitted; complete immediately if an empty array is passed

      Marble diagram:
      (an example is in the console)
    • merge

      Description:
      - Flattens multiple Observables together by blending their values into one Observable

      Marble diagram:
      (an example is in the console)
    • partition

      Description:
      - It's like filter, but returns two Observables: one like the output of filter, and the other with values that did not pass the condition

      Marble diagram:
      (an example is in the console)
    • race

      Description:
      - Race operator returns an observable, that when subscribed to, subscribes to all source observables immediately. As soon as one of the source observables emits a value, the result unsubscribes from the other sources

      Marble diagram:
      (an example is in the console)
    • zip

      Description:
      - Combines multiple Observables to create an Observable whose values are calculated from the values, in order, of each of its input Observables

      (an example is in the console)
  • Transformation Operators
    • buffer

      Description:
      - Collects values from the past as an array, and emits that array only when another Observable emits

      Marble diagram:
      (an example is in the console)
    • bufferCount

      Description:
      - Collects values from the past as an array, and emits that array only when its size reaches bufferSize
      - first argument is bufferSize: The maximum size of the buffer emitted
      - second argument is startBufferEvery: Optional. Default is null. Interval at which to start a new buffer. For example if startBufferEvery is 2, then a new buffer will be started on every other value from the source. A new buffer is started at the beginning of the source by default

      Marble diagram:
      (an example is in the console)
    • bufferTime

      Description:
      - Collects values from the past as an array, and emits those arrays periodically in time

      Marble diagram:
      (an example is in the console)
    • bufferToggle

      Description:
      - Collects values from the past as an array. Starts collecting only when opening emits, and calls the closingSelector function to get an Observable that tells when to close the buffer

      Marble diagram:
      (an example is in the console)
    • bufferWhen

      Description:
      - Collects values from the past as an array. When it starts collecting values, it calls a function that returns an Observable that tells when to close the buffer and restart collecting

      Marble diagram:
      (an example is in the console)
